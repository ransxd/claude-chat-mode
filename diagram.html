claude-chat-mode

Turn Claude Code into a conversational partner — zero source code changes

Architecture Overview
Layer 0 — User Input
Your Message
Every message enters the same Claude Code session. The routing layer below decides how Claude responds — as a conversational partner or a coding assistant.
Layer 1 — Routing
Three-Tier Mode Detection
CLAUDE.md Adaptive Mode auto-detects intent on every message.
/chat and /code slash commands give you an explicit manual override when auto-detection isn't enough.
Chat Mode
/chat
No tools Natural prose Warm tone Deep thinking

Replicate the claude.ai conversational experience inside Claude Code. Tools are suppressed unless explicitly requested. Responses flow in paragraphs, not bullet lists.
Code Mode
/code
All tools Concise Action-first Workflows

Standard Claude Code behavior. Read, Edit, Bash, Grep — everything active. Terse responses, immediate action on coding tasks.
Layer 2 — Output
Seamless Response
The user sees one continuous conversation. Mode switches are invisible — no banners, no announcements. Chat mode auto-exits when a coding task is detected, so you never get stuck.
File Structure
~/.claude/
  CLAUDE.md ← adaptive mode auto-detection
  commands/
    chat.md ← /chat slash command
    code.md ← /code slash command
